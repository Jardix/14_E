14.1 Exercise

1.) What is HTTP?

HTTP stands for Hypertext Transfer Protocol; it allows different machines to communicate across the web.

Answer:

Hyper Text Transfer Protocol is governs how clients get data from, or send data to, a server.

2.) What is a URL?

A URL is a 'nickname' for an address on the web.

Answer:

Short for Uniform Resource Locator, a URL is an address for some internet resource.

3.) What is DNS?

DNS is a naming system for machines connected to the internet.

Answer:

Short for Domain Name System, this is a system that takes human-readable URLs and converts them into IP addresses.

4.) What is a query string?

A query string is information used to modify the view of the webpage or resource you're requesting.

Answer:

The query string allows you to pass key-value pairs into the URL, in the format ?key1=value1&key2=value2...

5.) What are two HTTP verbs and how are they different?

GET and POST are two HTTP Verbs; 'GET' requests information from a server, POST gives information to that server for storage or modification.

Answer:

GET - get some data from the server (most pages, search forms) POST - send some data to the server (pages that change data on server)

6.) What is an HTTP request?

It is a task to be done on a resource or page. Searching a website, for example.

Answer:

An HTTP request is a request from a client to a server which follows the HTTP protocol (eg a request for HTML from news.google.com)

7.) What is an HTTP response?

A page, or other group of information given from the server to the machine that made the request.

Answer:

An HTTP response is a response from a server to a client which follows the HTTP protocol (eg sending back HTML/CSS/JS/etc)

9.) What are the processes that happen when you type “http://somesite.com/some/page.html” into a browser?

Your machine sends a 'GET" request to a server, that server may or may not send a request to a database, that server than sends the requested information back to your machine.

Anser:

Your browser “resolves” the name into an IP address using DNS Your browser makes a request to that IP address, including headers (info about browser, any previous cookies, and other things) The server sends a response (typically, HTML, with a status code (200 if it was sucessful) The browser makes a DOM from that HTML, and finds any other resources needed (images, CSS, JavaScript, etc) The browser makes separate HTTP requests for those resources and receives response from the server for each
